## 3、列表

**无序列表**：使用减号`-`作为列表标记，这些标记后面要添加一个空格：

- 第一项
- 第二项
- 第三项

**有序列表**：使用数字并加上句号`.`来表示，这些标记后面要添加一个空格：

1. 第一项
2. 第二项
3. 第三项

**列表嵌套**：可以创建多层次的结构，规则：
子列表需要缩进 2-4 个空格（推荐 2 个）；
保持一致的缩进长度；
可以无限层嵌套，但实际使用中建议不超过 3 层。

1. 准备阶段
   1. 收集资料
   2. 制定计划
2. 执行阶段
   1. 开始实施
   2. 监控进度
3. 总结阶段

- 水果
  - 苹果
    - 红苹果
    - 绿苹果
  - 香蕉
  - 橙子
- 蔬菜
  - 胡萝卜
  - 白菜

1. 第一项：
    - 第一项嵌套的第一个元素
    - 第一项嵌套的第二个元素
2. 第二项：
    - 第二项嵌套的第一个元素
    - 第二项嵌套的第二个元素

**任务列表**（复选框列表）：

- [x] 已完成的任务
- [ ] 未完成的任务
  - [x] 已完成的子任务
  - [ ] 未完成的子任务
- [ ] 另一个未完成的任务

## 4、引用块

引用块用于突出显示重要信息、引用他人观点或创建视觉层次。
Markdown区块引用是在段落开头使用`>`符号 ，然后后面紧跟一个空格符号。
> 区块引用
> 菜鸟教程
> 学的不仅是技术更是梦想

多级嵌套引用，一个`>`符号是最外层，两个`>`符号是第一层嵌套，以此类推。
> 最外层
> > 第一层嵌套
> > > 第二层嵌套

区块中使用列：
> 区块中使用列表
>
> 1. 第一项
> 2. 第二项
>
> - 第一项
> - 第二项
> - 第三项

列表中使用区块，需要在`>`前添加四个空格的缩进：

- 第一项
    > 菜鸟教程
    > 学的不仅是技术更是梦想
- 第二项

引用块内可以包含几乎所有其他Markdown元素。
成功提示：
> &#x2705; **成功**
> 配置已保存并生效。系统将在下次重启时应用新设置。

警告信息：
> &#x26a0;&#xfe0f; **警告**
> 此操作不可逆转，请确保已备份重要数据。

错误信息：
> &#x274c; **错误**
> 连接数据库失败，请检查网络连接或联系系统管理员。

信息提示：
> &#x2139;&#xfe0f; **提示**
> 首次使用需要进行账户验证，验证邮件已发送到您的邮箱。

重要提示：
> **重大变更**
>
> &#x26a0;&#xfe0f; API 接口路径已调整，旧版本客户端需要更新
>
> 详见 [迁移指南](./migration-guide.md)

## 5、代码

Markdown提供了多种方式来展示代码，从简单的行内代码到复杂的代码块，满足不同场景下的代码展示需求。

**行内代码**：段落上的一个函数或片段的代码可以用反引号`` ` ``把它包起来，如：`printf()`函数。

**缩进式代码区块**：代码区块使用4个空格或者一个制表符（Tab键）。
正常文本段落

    这是缩进式代码块
    每行前面有四个空格
    保持代码的原始格式，如：
    module xor(
        input         a,
        input         b,
        output        c
    );

继续正常文本（注意代码块前后有空行）。

**三反引号代码块**：用```` ``` ````包裹一段代码，并指定一种语言（也可以不指定）。
正常文本段落

```verilog
多行代码内容
可以包含空行
保持原有缩进
module xor(
    input         a,
    input         b,
    output        c
);
```

继续正常文本（注意代码块前后有空行）。

**行号显示**：通过在语言后加上`{.line-numbers}`实现。

```javascript {.line-numbers}
function fibonacci(n) {
    if (n <= 1) return n;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

console.log(fibonacci(10));
```

**代码差异对比**：用于显示代码的添加、删除或修改，常用于展示版本控制中的变更。

```diff
function calculateTotal(items) {
-   let total = 0;
+   let total = 0.0;
    
    for (let item of items) {
-       total += item.price;
+       total += parseFloat(item.price);
    }
    
+   // 保留两位小数
+   total = Math.round(total * 100) / 100;
    return total;
}
```

- [第二章：安装](#第二章安装)

## 8、表格

表格能够清晰地展示结构化数据。
制作表格使用`|`来分隔不同的单元格，使用`-`来分隔表头和其他行。

| 默认 | 左对齐 | 右对齐 | 居中对齐 |
| ---- | :---- | ----: | :----: |
| 单元格 | 单元格 | 单元格 | 单元格 |
| 单元格单元格 | 单元格单元格 | 单元格单元格 | 单元格单元格 |

| 功能 | 描述 | 状态 |
| ---- | ---- | :---: |
| **用户登录** | 支持邮箱和手机号登录 | &#x2705; |
| *密码重置* | 通过邮箱重置密码 | &#x26a0;&#xfe0f; |
| `API接口` | RESTful API 设计 | &#x2705; |
| [文档链接](https://example.com) | 查看详细文档 | &#x1f4d6; |

| 项目 | 详细说明 |
| --- | -------- |
| 需求分析 | 1. 收集用户需求<br>2. 分析业务场景<br>3. 确定功能范围 |
| 技术选型 | 前端：React + TypeScript<br>后端：Node.js + Express<br>数据库：MongoDB |
